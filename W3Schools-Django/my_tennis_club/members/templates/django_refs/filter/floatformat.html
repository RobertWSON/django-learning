Definition and Usage

The floatformat filter rounds a floating-point number to a specified number of decimals, or one, if no decimal argument is specified.

If argument decimal value is a positive number, a value will be displayed with specified number of decimals, even if value is an integer:

{#

Floatformat with Positive Decimal Number Example

try larger heading#}

Floatformat with Positive Decimal Number Example

{#{% include 'django_refs/filter/floatformat_example2.html' %} Do not need to use this because example is #} {# within same HTML Template page #}

{{ 7.122489|floatformat:2 }}

{{ 7.1|floatformat:2 }}

{{ 7|floatformat:2 }}

As you can see from this floatformat example, we are using a django command to round each of 3 numbers above to 2 positive decimal places. 7.122489 and 7.1 are floating point numbers and 7 is an integer that still gets rounded to 2 positive decimal places.

Syntax

try larger heading#}

Syntax

{ { value|floatformat:decimalsg } }

Template filters are defined by using a pipe | character followed by name of filter.

Arguments are defined by using a colon : character followed by argument value(s).

{#

Arguments

try larger heading#}

Arguments

{##} {##} {##}
Value Description
decimalsdecimals Optional. A number specifying number of decimals. Default value is -1,
which will round a number to only one decimal, or no decimals if number
has no decimals.
gg Optional. If present, a number will be displayed with a seperator for each
thousand, using local settings (using , for English and . for Norwegian).
uu Optional. If present, a number will be displayed with a seperator for each
thousand, using local settings. If u is present g cannot be present.

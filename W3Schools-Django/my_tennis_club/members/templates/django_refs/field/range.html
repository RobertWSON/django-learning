Field Lookups - range

Range Example with Numbers

Get all records where id is between 2 and 4 :

{# Range filter Example with numbers, starts here #}

The queryset object:

{{ mymembers_numeric }}

Loop through the items:

{% for x in mymembers_numeric %} {% endfor %}
ID Firstname Lastname
{{ x.id }} {{ x.firstname }} {{ x.lastname }}
{# Range filter Example with numbers, ends here #} {# Image showing views.py file, so we can see how range works with data for this numeric range example #} {# #}

Here we have mymembers_numeric variable, which is an object and we are using this with a name of mydata_numeric outside of context.
After defining our view, we are using mydata_numeric to access Member table to get all objects (records).
For all records, we are getting all values for fields and using a numeric range filter to show only records that have an id from 2 to 4.
id has a numeric value for identifying a record and this is why we call it a numeric range filter.

{# Image showing HTML Template code, so we can see how data is used for this numeric range example #} {# #}

In HTML Template we are using a for loop with mymembers_numeric object from views.py file.
After using numeric range filter in views.py, we are showing all field values that we want, for records with an id from 2 to 4.
We are using mymember_numeric surrounded by double curly braces, to show a queryset on page with details of those filtered records.
You will notice in queryset, that filtering by id, shows all field values and joined_date field uses a datetime function to show date that
a member joined tennis club.

Definition and Usage

The range lookup is used to get records that are between two values.

A field can be of any type, that can either be numeric, string or dates.

Range Example with Alphanumeric

Alphanumeric means that this range filter can be used on strings, numbers and dates.

{# Range filter Example with alphanumeric characters, starts here #}

The queryset object:

{{ mymembers_alphanum }}

Note: When searching in text fields, remember that 'G' has a lower alphanumeric value than 'g' .

SQL Equivalent

An SQL Equivalent for two examples above will be :

Numeric example
WHERE id BETWEEN 2 AND 4

Alphanumeric example
WHERE id BETWEEN 'G' and 'M'

Syntax

All Field lookup keywords must be specified with fieldname, followed by two underscore characters __ and a keyword:

fieldname__range=value

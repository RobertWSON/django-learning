Here we have mymembers variable, which is an object and we are using this with a name of mydata, outside of context.
After defining our view, we are using mydata to access Member table to get all objects (records).
For all records, we are getting all values for fields and using an exact filter to show a record or records that have an exact matching firstname field.

In HTML Template we are using a for loop with mymembers object from views.py file.
After using exact filter within views.py, we are showing field values that we want, for records with a firstname field that matches our exact filter.
We are using mymembers, surrounded by double curly braces, to show a queryset on page with details of our filtered record.
You will notice in queryset, that joined_date field uses a datetime function to show date that a member joined tennis club.

Definition and Usage

The exact lookup is used to get records with a specified value.

The exact lookup is case sensitive.

For a case insensitive search, use the iexact lookup.

SQL Equivalent

The SQL equivalent to the example above will be:

WHERE firstname = 'Emil';

Syntax

All Field lookup keywords must be specified with fieldname, followed by two underscore characters __ and a keyword:

fieldname__exact='value'

Autoescape On with Characters Example


Here we are demonstrating how Django Autoescape On works for a heading variable that has characters with HTML entities.
In our views.py, just like we did with Autoescape Off, we have same heading_chars variable Hello &lt;i&gt;my&lt;/i&gt; World!
and this is how it shows on our web page below.

{% autoescape on %}

{{ heading_chars }}

{% endautoescape %}

When Django Autoescape was off with same heading_chars variable, we saw that it showed italic tags as text with heading.

Now with Django Autoescape on with characters, you can see that it shows same heading_chars variable from our views.py file,
as it does on our web page.
This is because HTML entities in our variable are already escaped and do not need to be converted or changed to text characters.
This is how Django Autoescape On behaves when we have a variable with HTML entities.

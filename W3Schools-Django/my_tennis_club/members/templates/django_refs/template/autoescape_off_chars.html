Autoescape Off with Characters Example


Here we are demonstrating how Django Autoescape Off works for a heading variable that has characters with HTML entities.
In our views.py we have a heading_chars variable Hello &lt;i&gt;my&lt;/i&gt; World! and this is how it shows on
our web page below.

{% autoescape off %}

{{ heading_chars }}

{% endautoescape %}

You can see how italics HTML tags get shown as text and are not recognised as HTML, when autoescape off is used in this way.

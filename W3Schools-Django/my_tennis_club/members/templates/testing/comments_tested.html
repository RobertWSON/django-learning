Django Comment Tag

Comments

Comments allow you to have sections of code that should be ignored.

Comment Example

{% comment %} first section is normal html code {% endcomment%}

Welcome Everyone

{% comment %}

Welcome ladies and gentlemen

Here we are commenting html h1 heading code. Allowing a section of code to be ignored {% endcomment %}

Comment Description

You can add a message to your comment, to help you remember why you wrote comment, or as a message to other people reading code.

Comment Description Example

Add a description to your comment:

{% comment %} first section is normal html code {% endcomment%}

Welcome Everyone

{% comment %} Here we are adding a message to your comment below, to help you remember why you wrote the comment, or as a message to other people reading the code.{% endcomment %} {% comment "this was the original welcome message" %}

Welcome ladies and gentlemen

{% endcomment %}

Smaller Comments

You can also use { # ... # } tags when commenting out code, which can be easier.
for smaller comments:

Smaller Comments Example

Comment out the word Everyone:

{% comment %} You can also use Curly Braces with Hash tags inside it for commenting out code, which can be easier for smaller comments {% endcomment %} {# In this h1 heading code below, Everyone has been commented, so it does not show on localhost webpage #} {# and only Welcome gets shown on localhost web page. #}

Welcome{# Everyone #}

Django 404 (page not found)

Page Not Found

A 404 error is a message that appears on a web page when it cannot be found.
If you try to access a page that does not exist, Django directs you to a built-in view that handles 404 errors and show a 404 error on a web page.

You will learn how to customize this 404 later on this page, but for now just try to request a page that does not exist.

In browser window, type 127.0.0.1:8000/masfdfg in address bar.

Customize Django 404 Page

As we have seen, Django has a built-in template that only shows "Not Found" on a web page when it cannot find a file for a web page.
Showing "Not Found" is a default behaviour of Django, where it looks for a file named 404.html in a templates folder
and shows a 404 error with "Not Found" when a page is not found or does not exist.

However, we can customize a 404 error message to have it show what we want on a page that has not been found or does not exist.
To do this we have to create a file in templates folder, name it 404.html and fill it with what you want to show on a web page as an error.

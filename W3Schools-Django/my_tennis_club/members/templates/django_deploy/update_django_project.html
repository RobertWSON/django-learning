Update Django Project

Deploy Changes

Any changes you do with application locally on your own computer are not visible on deployed
version.

So if you make a change and you want it to be visible on deployed version, you have to upload a
new .zip file.

Create a .zip File

To wrap your project into a .zip file, follow steps from Create .zip File section:

Start by selecting relevant fles and folders of your project. If you have been following this project build,
you should select highlighted files in example below:

{# #}

Right-click on selected files and choose create a .zip file.

Now you have a .zip file containing project changes and you can upload it to Elastic Beanstalk:

{# #}

Upload to Elastic Beanstalk

Log into your Amazon Web Services Account, find your project under "Elastic Beanstalk" application
and click upload and deploy:

{# #}

Choose .zip File

Click "choose file" button to upload .zip file created just before:

{# #}

Deploy

Click "Deploy" button:

{# #}

Uploaded!

That's it, your project is updated with all new changes, Awesome.

Note: Follow these steps every time you want to update your project.

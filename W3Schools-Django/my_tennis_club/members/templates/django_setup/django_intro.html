Django Introduction

What is Django ?

Django is a mainly a back-end server side web development framework that is written in Python and it has
features that allow it to handle frontend web development as well.

For those that are not familiar with difference between frontend and backend web development, I will make it clear.

Frontend web development is code used to display what is on a web page, so you can see it on your screen.
Backend web development is code used to make a connection with a server and a common task of getting data from a database with Django.

Django makes it easier to create websites because it takes care of difficult stuff, so that you can concentrate on
building your web applications.

Django emphasizes reusability of components, which is referred to as DRY (Don't repeat yourself) and comes
with ready-to-use features like login system, database connection and CRUD operations (Create Read Update Delete).

It must be remembered that Django is especially helpful for database driven websites, it is free and open source, so code is available to see for public.

Django History

Django was invented by Lawrence Journal-World in 2003 to meet short deadlines in newspapers and at same
time meeting demands of experienced web developers.

Initial release to public was in July 2005.

Latest version of Django is 5.0.2 (February 6, 2024)

How does Django Work ?

Django follows a Model View Template design pattern (MVT).

  • Model - This is data that you want to present and is usually data from a database.
  • View - A request handler that returns a revelant template and content - base on a request from a user.
  • Template - A text file (usually an HTML file) that has layout of a web page, with logic on how to display data.

Django Model

A Django model provides data from a database.

In Django, data is delivered as Object Relational Mapping (ORM), which is a technique designed to make it easier to work with databases.
ORM converts data between a relational database and objects in an object oriented programming language.

Most common way to extract (get) data from a database is SQL, but one problem with this is that you have to have
a pretty good understanding of a database structure to be able to work with it.

Django with ORM, makes it easier to communicate with a database, without having to write complex SQL statements.

Models are created in a file called models.py .

Django View

A view is a function that takes http requests as arguments, imports relevant model(s), finds out what data to
send to template and returns a final result.

Views are created in a file called views.py .

Django Template

A template is a file (it can be called a text document), where you describe how a result should be displayed.

Templates with Django are often .html files, with HTML code describing layout of a web page.
As a Template is a text document, it can also be in other file formats to show other results, but for this website
we will concentrate on .html files.

Templates of an application are located in a folder named templates .

Django URL

Django provides a way to navigate around different web pages in a website through a URL.

When a user requests a URL, Django decides which view it will send it to and this is done in a file called urls.py .

What is Happening ?

When you have installed Django, created your first web application and browser requests a URL, this is what
happens :

  1. Django receives a url, checks urls.py file and calls a view that matches a URL .
  2. A view in views.py file checks for relevant models.
  3. Models are imported from models.py file .
  4. A view sends data to a specified template in a template folder .
  5. A template contains HTML and Django tags and with data it returns finished HTML content back to a browser.

This is basically what you will learn in this Django learning website and are basic steps in making a Django web application.
We will learn what an application is later.

Django can do more than this, but what we learn is enough to be able to produce a stylish website.
